// Описание диаграммы БД TXSTBXRD.
Схему создавал тут - https://dbdesign.online/

% Таблица "Project" || Содержит описание Тестового Проекта.
1. ID - первичный ключ.
2. Name - Имя проекта.
3. ScreenSaver - Шапка или фон проекта.
4. LastRun - Дата и время последнего прогона (Теста или группы тестов).
5. ItemCount - Количество доступных тестов.
6. GroupCount - Количество груп тестов.


% Таблица "Consumers" || Содержит описание Работника у которого есть доступ к пространству проекта.
1. ID - первичный ключ.
2. Name - Имя.

% Таблица "Project_Consumers" || Реализует связь многие ко многим, между таблицами проект и работник.
1-2. первичные ключи одноименных таблиц.
3. StartDate - Дата добавления работника к проекту.
4. EndDate - Дата удаления работника из проекта.

% Таблица "Item" || Содержит описание теста.
1. ID - первичный ключ.
2. Name - Название теста.
3. Result - Результат выполнения.
4. ResultDescription - Описание результата выполнения теста.
5. LastRun - Дата и время последнего запуска теста.
6. GroupId - Вторичный ключ, ссылающийся на группу.

% Таблица "GroupItems" || Содержит описание группы тестов.
1. Id - первичный ключ.
2. Name - название группы тестов.
3. ProjectId - Вторичный ключ, ссылающийся на проект.
4. ItemCount - Количество тестов в группе.
5. LastRun - Последний прогон группы или тестов внутри группы.
6. Default - Являеться ли она группой по умолчанию, нужно для реализации корневых тестов.
